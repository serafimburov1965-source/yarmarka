# Ярмарка — запуск за 20 минут

## 1. База данных (Supabase, бесплатно)

1. Зайди на **supabase.com** → Sign up (можно через GitHub)
2. New Project → придумай имя и пароль → дождись создания (1-2 минуты)
3. Слева в меню открой **SQL Editor** → New query
4. Скопируй туда содержимое файла `supabase.sql` из этого проекта → Run
5. Слева открой **Project Settings → API**
6. Скопируй **Project URL** и **anon public key** — они понадобятся дальше

## 2. Выложить код на GitHub

1. Зайди на **github.com**, создай новый репозиторий (например `yarmarka`)
2. Загрузи туда все файлы из этой папки (`yarmarka-app`) — можно перетащить прямо в браузере через "uploading an existing file"

## 3. Деплой на Vercel (бесплатно)

1. Зайди на **vercel.com** → Sign up через GitHub
2. **Add New → Project** → выбери свой репозиторий `yarmarka`
3. В разделе **Environment Variables** добавь:
   - `VITE_SUPABASE_URL` = тот URL из шага 1.5
   - `VITE_SUPABASE_ANON_KEY` = тот anon key из шага 1.5
4. Нажми **Deploy** → через минуту получишь ссылку вида `https://yarmarka.vercel.app`

## 4. Подключить к Telegram-боту

1. Открой **@BotFather** в Telegram
2. `/mybots` → выбери своего бота → **Bot Settings → Menu Button → Configure menu button**
3. Пришли ссылку с Vercel (`https://yarmarka.vercel.app`)
4. Пришли название кнопки (например «Ярмарка»)
5. Открой своего бота — рядом со строкой ввода появится кнопка меню, которая открывает приложение

## Важно про безопасность

Токен бота (из BotFather) нигде в этом коде не используется — код фронтенда его не знает и знать не должен. Держи токен в секрете, никому не показывай и не вставляй в код.

Таблица в Supabase сейчас открыта на запись всем (см. `supabase.sql`) — это ок для MVP и теста, но перед реальным запуском с деньгами стоит вернуться и настроить более строгие правила доступа.

## Локальный запуск для тестов

```
npm install
cp .env.example .env   # и вписать туда свои ключи Supabase
npm run dev
```
